Fountain Coach

a teatro-style reasoning interface

Fountain.coach is not a startup. It’s not a product. It is a semantic system — developed slowly, through rhythm, resistance, and repetition.

Like a rehearsal script passed between performers, its pages grew not through planning but through attention. Attention to gesture. To contradiction. To what returns when nothing is forced.

At its technical core lies FountainAI — a modular architecture of roles and invocations. Each service speaks its line through an OpenAPI script. Each route a gesture. Each model a mood.

The structure is not technological. It is dramaturgical. It asks not:
How do we automate this?
It asks: How does this speak when placed onstage?

Fountain defines its roles not by what they do, but by how they respond — to presence, to absence, to rhythm, to change.

There are no dashboards. No onboarding flows. Only roles. Relations. Resonance.

When something breaks in Fountain, it does not fail. It reenters. It begins a new scene.

The observer becomes the performer. The user becomes the ensemble. The API becomes the monologue.

This is not prompt engineering. It is semantic direction. This is not UI design. It is composition. This is not software. It is practice.
